# Kaggle dataset → our database (mapping)

**Single reference** for dataset files and seed data (products, orders, init scripts).  
The files in **dataset/kaggle/** come from [Kaggle – Coffee Shop Sales/Inventory/Staff](https://www.kaggle.com/datasets/viramatv/coffee-shop-data). This document describes how they map to our schema and what was done.

---

## Files in dataset/kaggle/

| File | Contents |
|------|----------|
| **items.csv** | Products: item_id, sku, item_name, item_cat, item_size, item_price |
| **orders.csv** | Transactions: order_id, created_at, item_id, quantity, cust_name, in_or_out |
| **inventory.csv** | Ingredients stock: inv_id, ing_id, quantity (not product-level) |
| **ingredients.csv** | Ingredients master: ing_id, ing_name, ing_weight, ing_meas, ing_price |
| **recipe.csv** | Recipe: recipe_id (sku), ing_id, quantity (links items to ingredients) |
| **staff.csv**, **shift.csv**, **rota.csv** | Staff/scheduling (not used in our e-shop schema) |

## What we did with the dataset

### Products (done)

- **Source:** `items.csv` (24 rows).
- **Mapping used:**
  - `item_name` + `item_size` (if not N/A) → **products.name** (e.g. "Cappuccino (Medium)", "Espresso")
  - `item_cat` → **categories.category_name** and **products.category** (Hot Drinks, Cold Drinks, Snacks)
  - `item_price` → **products.price** (kept as in dataset, e.g. 3.45, 5.60)
  - Short description from category/size → **products.description**
  - Default **products.stock_quantity** = 50 (dataset has no product-level stock)
  - **products.image_url** = placeholder (https://picsum.photos/seed/{item_id}/400/400); replace with real images if you like.

- **Output:** **product-service/src/main/resources/product-seed.json** was generated from this mapping. When product-service starts with an empty `products` table, it loads this file and inserts 24 products and 3 categories (Hot Drinks, Cold Drinks, Snacks).

### Orders / order_items (done)

- **orders.csv** has one row per line item: order_id, created_at, item_id, quantity, cust_name, in_or_out.
- **Mapping to our DB:**
  - Distinct **order_id** → **orders** (script uses a single seed **user_id = 1**; the script inserts that user if missing).
  - **created_at** → **orders.created_at** (and **orders.total_amount** = sum of item price × quantity per order).
  - **item_id** (It001–It024; It0010 → product 10) → **order_items.product_id** (product id 1 = It001, … 24 = It024). Rows with It025–It028 (not in items.csv) are skipped.
  - **quantity** → **order_items.quantity**; unit **price** from items.csv → **order_items.price**.

- **Import script:** **tools/generate-orders-seed.js** reads **orders.csv** and **items.csv**, groups by order_id, and writes **init-db/07-seed-orders-from-dataset.sql**. Run it with: `node tools/generate-orders-seed.js` (from project root). Then run **init-db/07-seed-orders-from-dataset.sql** in MySQL **after** 06-seed-products-from-dataset.sql (so products 1–24 exist). The 07 script inserts a seed user (id=1) if needed, then orders and order_items.

## Summary

- **Products:** Built from **items.csv** and stored in **product-seed.json**. Auto-imported on first run of product-service, or via **06-seed-products-from-dataset.sql**.
- **Categories:** Hot Drinks, Cold Drinks, Snacks (from item_cat).
- **Orders:** Imported via **07-seed-orders-from-dataset.sql** (generated by `node tools/generate-orders-seed.js`). Run 07 after 06 so products 1–24 exist.
- **Images:** Placeholder URLs; replace in the JSON or in the DB (or via API) with real image URLs.

All product and order data in the app (MySQL, API, React) can come from this Kaggle dataset.

---

## How the data gets into the SQL database

You have **two ways** to get the 24 products and 3 categories into MySQL:

1. **Run the SQL scripts (no Java needed)**  
   In MySQL Workbench (or command line), run **init-db/01-single-database-all-tables.sql** first (if you haven’t already), then **init-db/06-seed-products-from-dataset.sql** (categories + 24 products), then **init-db/07-seed-orders-from-dataset.sql** (seed user if needed, orders + order_items from orders.csv). Run 07 after 06 so products 1–24 exist.

2. **Let the app do it**  
   If the `products` table is **empty**, when you start **product-service** (with profile `mysql`), it loads **product-service/src/main/resources/product-seed.json** and inserts the same 24 products and 3 categories automatically.

Use (1) if you want the data in the database before starting the app; use (2) if you prefer the app to seed on first run.
